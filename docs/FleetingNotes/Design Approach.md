# Design Approach
Created : 05-04-2023 08:50

There are two kinds of #design-approach. The #top-down approach and the #bottom-up approach. While both seem to have their advantages and disadvantages, the aim is to use them simultaneously.

Notes based on research via ChatGPT, for the #facade industry

## Top Down Approach
* Usually how a Designer / Architect starts working.
* Start with the big picture / entire building.
* Considers the #Site, #SiteContext, Cultural fit, Urban impact, Visual / Architectural Style, etc. to arrive at design parameters.
* Once the overall concept has been decided,  attention shifts to one level of details.

## Bottom Up Approach
* Usually when there are constraints on cost, material, regulations, and/or construction techniques, the focus is first on these details.
* The overall idea is then built up step by step.
* When there is a framework to work within, Bottom Up approach is preferred.

## Combining both approaches
1. Single Switch
   Start with Top-Down to define high-level parameters. Then switch to Bottom-Up approach to work with details, and constraints.
